Halborn Added as CVE Numbering Authority (CNA)

newsApril 11, 2023

Halborn is now a CVE Numbering Authority (CNA) for all blockchain and Web3 products that rely on smart contracts written in Rust, Go, and Solidity, as well as blockchain associated Web2 and Web3 infrastructure not covered by another CNA.

VulnCheck Added as CVE Numbering Authority (CNA)

newsApril 11, 2023

VulnCheck is now a CVE Numbering Authority (CNA) for vulnerabilities discovered by, or reported to, VulnCheck that are not in another CNA’s scope.

CVE Program Booth at RSA 2023 on April 25-27, 2023

newsApril 11, 2023

The CVE Program will host a CVE Booth (#6566) on April 25-27, 2023, at RSA Conference 2023 at the Moscone Center in San Francisco, California, USA.

Schweitzer Engineering Laboratories Added as CVE Numbering Authority (CNA)

newsApril 4, 2023

Schweitzer Engineering Laboratories, Inc. is now a CVE Numbering Authority (CNA) for all Schweitzer Engineering Laboratories products.

StrongDM Added as CVE Numbering Authority (CNA)

newsApril 4, 2023

StrongDM is now a CVE Numbering Authority (CNA) for StrongDM issues only.

Help Shape the CVE Record/ID Search Capability on the CVE.ORG Website – 3 Question Survey

newsApril 4, 2023

The CVE Program is conducting research about how you use and/or would like to use CVE ID and CVE Record search on the CVE website with a three-question survey. Your responses will help shape future search capabilities for the cve.org website.

Minutes from CVE Board Teleconference Meeting on March 15 Now Available

The CVE Board held teleconference meeting on March 15, 2023. Read the meeting minutes.

Now Available — CVE List Downloads in CVE JSON 5.0 Format

blog

By CVE Program |

March 29, 2023

CVE Records in CVE JSON 5.0 format are now available for bulk download by the community. “Bulk download” means all CVE Records and updates are included in a single download file.

CyberDanube Added as CVE Numbering Authority (CNA)

newsMarch 28, 2023

CyberDanube is now a CVE Numbering Authority (CNA) for all CyberDanube products, as well as vulnerabilities in third-party hardware/software discovered by CyberDanube that are not in another CNA’s scope.

Securifera Added as CVE Numbering Authority (CNA)

newsMarch 28, 2023

Securifera, Inc. is now a CVE Numbering Authority (CNA) for vulnerabilities in vendor products discovered by Securifera, or related parties, while performing vulnerability research or security assessments.
